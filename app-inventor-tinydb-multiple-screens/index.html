The cross-screen rule

Shared data still needs an exact name.

TinyDB can preserve a value while your app changes screens. The second screen only retrieves that value when GetValue asks for the exact tag StoreValue wrote.

01

Treat every tag as an exact address.

App Inventor does not treat similar-looking literal tags as the same address. A change in case, spacing, punctuation, or spelling points GetValue somewhere else.

  • profile_name is not profile-name
  • PlayerScore is not playerscore
  • user name is not username

Choose one canonical tag, then use it everywhere that value is stored or retrieved.

Screen1 / StoreValue / profile_name Screen2 / GetValue / profile_name

Rename and retest

A five-step TinyDB repair.

Change one thing at a time. That keeps a high-confidence tag fix separate from defaults, event order, or other block behavior you may need to inspect later.

  1. 01

    Store one known test value on the first screen

    Use a simple value you can recognize, then follow the same screen path that currently fails.

    Known starting point
  2. 02

    Copy the literal StoreValue tag exactly

    Note every character, including uppercase letters, spaces, hyphens, and underscores.

    Canonical tag
  3. 03

    Compare the GetValue tag on the second screen

    Search all blocks involved in that data flow. Similar-looking text is not enough. The literal tag must match exactly.

    Mismatch found
  4. 04

    Rename only the mismatched tag

    Use the canonical spelling everywhere that value is stored or retrieved. Keep other logic unchanged for this test.

    One controlled repair
  5. 05

    Store fresh data and retest both screens

    Rerun the same path with a new value. If the exact tags now match but retrieval still fails, move to event order, dynamic-tag, and default-value checks manually.

    Repair verified

Quick answers

TinyDB multiple screens questions.

Short answers for the decisions that usually slow down this repair.

Can App Inventor TinyDB work across multiple screens?

Yes. A stored TinyDB value can persist while the app changes screens. The next screen must ask for the same tag text that was used when the value was stored.

Why does Screen2 keep showing the default value?

GetValue uses its valueIfTagNotThere input when the requested tag has no saved value. Compare the tag text first, then confirm that StoreValue runs before Screen2 retrieves it.

Does the analyzer upload my App Inventor project?

No. The analyzer reads the exported .aia locally in your browser. It does not send the project file or its contents to a server.
