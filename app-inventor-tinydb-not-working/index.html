Three clues to check first.

"TinyDB not working" can describe several different failures. Start with the visible symptom, then test one cause at a time.

01 / CROSS-SCREEN

The value works on Screen1, then vanishes on Screen2

Compare the StoreValue tag on the first screen with the GetValue tag on the second. Case, punctuation, spacing, and spelling must match exactly.

02 / RETRIEVAL

GetValue keeps showing the fallback

Confirm StoreValue actually runs before retrieval. A fallback also appears when GetValue asks for a tag that has never been stored.

03 / OLD DATA

An older value keeps coming back

Search for every block that writes the same tag. A second StoreValue block can replace your newer value later in the flow.

    Store a fresh value, navigate through the same screen order, and retrieve it again. If the literal tags now match but the bug remains, move to the manual checks below.

TinyDB debugging questions.

Short answers for the points that usually slow down a cross-screen repair.

Why does TinyDB work on Screen1 but not Screen2?

First compare the exact StoreValue and GetValue tag text on both screens. A change in case, punctuation, spacing, or spelling creates a different tag, so Screen2 can return its fallback instead of the value you stored.

Why does GetValue keep returning the default?

GetValue returns its valueIfTagNotThere input when the requested tag has no saved value. Confirm that StoreValue runs before retrieval and that both blocks use the same tag text.

Does the analyzer upload my .aia project?

No. The analyzer reads your exported .aia locally in the browser. It does not send the project file or its contents to a server.
